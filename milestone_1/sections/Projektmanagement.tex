\section{Projektmanagement}
%
\subsection{Kontext}
Unser Projekt für die VU ist \emph{Offene Daten erlebbar machen: Mobile App zur interaktiven Visualisierung von Open Data der Stadt Wien}. Daher haben wir uns bei der Problemanalyse stark auf das Thema Datenvisualisierung konzentriert.\\
\noindent
Im Rahmen der VU wird ein App-Prototyp erstellt, mit allgemeinen und themenspezifischen Rahmenbedingungen.
%
\subsection{Motivation und Ziele}
Unsere Motivation und unser Ziel sind durch die Problembeschreibung gut zusammengefasst.
Abseits davon:
%
\begin{itemize}
    \item eine stabile, schöne und funktionierende App erstellen
    \item mit guter UI / UX
\end{itemize}
%
\subsection{Nichtziele}
%
\begin{itemize}
    \item von der Problembeschreibung abweichen
    \item eigene Daten generieren
    \item eine unsichere App erstellen
    \item etwas Langweiliges, Uninteressantes kreieren
    \item schlechtes UI / UX
\end{itemize}
%
\section{Schwerpunkte und Lernwünsche}
Da unsere Gruppe aus Informatik-Studenten besteht, wird der programmiertechnische Teil nicht allzu schwierig sein. Als Schwerpunkt und Lernwunsch haben wir also vor, unsere Fähigkeiten, UX/UI zu designen und zu implementieren.
